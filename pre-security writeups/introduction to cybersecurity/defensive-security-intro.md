## Summary

Introduction to the blue team side of cybersecurity. Covers what defensive security means, how Security Operations Centers work, and the core disciplines defenders use — from threat intelligence to malware analysis and incident response.

---

## Key Concepts Learned

- **Defensive security** — Protecting systems by detecting, responding to, and recovering from threats; the opposite of offensive hacking
- **Security Operations Center (SOC)** — A team that monitors an organization 24/7, triages alerts, investigates incidents, and escalates serious events
- **Threat intelligence** — Knowing your adversaries' tactics, tools, and motives so you can defend proactively rather than just react
- **DFIR (Digital Forensics & Incident Response)** — When a breach happens, DFIR teams investigate what happened, preserve evidence, and contain the damage
- **Malware analysis** — Examining malicious software to understand what it does, how it spreads, and how to stop it

## Notes & Walkthrough

### Task 1 — Introduction to Defensive Security

Defensive security focuses on two main tasks: preventing intrusions and detecting/responding when they occur. Unlike offensive security (finding vulnerabilities), defensive security is about monitoring, alerting, and responding.

### Task 2 — Areas of Defensive Security

Key areas covered:
- **SOC** — monitors network and systems 24/7 using tools like SIEM
- **Threat Intelligence** — collects data on threat actors to anticipate attacks
- **DFIR** — responds to confirmed incidents, performs forensics, and writes reports
- **Malware Analysis** — static and dynamic analysis to understand malicious code behavior

### Task 3 — Practical Exercise

Worked through a simulated SOC scenario — reviewing alerts, identifying malicious activity, and understanding the triage workflow.
