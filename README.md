## What is TryHackMe?

TryHackMe is a browser-based cybersecurity training platform built around guided, hands-on learning paths. Each path is made up of rooms — interactive labs combining theory, practical exercises, and real tools running directly in the browser with no local setup required.

It covers:

- **Pre-Security (SEC0)** — networking, Linux, Windows, and web fundamentals
- **Cyber Security 101 (SEC1)** — foundational security concepts
- **SOC Level 1 (SAL1)** — blue team operations, SIEM, log analysis, and incident response
- **Jr Penetration Tester** — offensive security techniques and methodology

TryHackMe is the **primary learning platform** for Phase 1 and Phase 2 of my roadmap.

## Skills Gained So Far

### 🌐 Networking

- IP and MAC addressing — logical vs hardware-level identification
- DNS — record types (A, AAAA, CNAME, MX, TXT), resolution flow, TTL, caching, spoofing risks
- ICMP / ping — connectivity testing and what it looks like in logs
- LAN topologies — Star, Bus, Ring; switches vs routers; ARP and DHCP
- ARP — IP-to-MAC resolution; ARP spoofing as a MitM vector
- DHCP — IP assignment automation; rogue DHCP attacks
- Subnetting — network division for security, performance, management
- OSI Model — all 7 layers, encapsulation/de-encapsulation, protocols per layer
- Layer-to-attack mapping — Layer 7 (web), Layer 4 (SYN floods), Layer 2 (ARP spoofing)
- Packet structure — headers, frames, encapsulation across layers
- Extending networks — port forwarding, VPN, wireless fundamentals
- Core protocols — TCP, UDP, ICMP, FTP, SMTP behaviors
- Secure protocols — TLS, SSH, VPN — what they protect and how

### 🌐 Web Fundamentals

- HTTP methods, headers, status codes, cookies, sessions
- How frontend (HTML/CSS/JS) and backend connect
- DNS + HTTP + hosting integration — full request lifecycle

### 🐧 Linux

- Filesystem hierarchy, basic commands, navigation
- SSH access, file operations, permissions
- Common utilities, package management
- Shell basics for day-to-day analyst work

### 🛡 Defensive Security

- SOC operations — 24/7 monitoring, alert triage, escalation workflows
- DFIR — incident investigation, evidence preservation, containment, reporting
- Threat intelligence — adversary tactics, tools, motives
- Malware analysis — static and dynamic approaches

### ⚔️ Offensive Security

- Ethical hacking mindset — attacking systems legally to expose weaknesses
- Web application reconnaissance with `gobuster`
- Why offensive knowledge makes you a better defender

### 💻 Foundations

- Virtualisation — why it powers modern IT and security work
- Cloud computing fundamentals
- Computer architecture and OS introduction

### 💼 Career & Context

- Clear understanding of the IR/DFIR Analyst role and day-to-day responsibilities
- How Project Management skills transfer to incident response coordination
- Certification roadmap: SEC0 → SEC1 → SAL1 → Security+ → CDSA → eCIR

## Tools Introduced

| Tool | Command | Purpose |
| --- | --- | --- |
| `ping` | `ping <target>` | Test reachability via ICMP |
| `gobuster` | `gobuster dir -u <url> -w <wordlist>` | Brute-force web directories |
| `nslookup` | `nslookup <domain>` | Query DNS records |
| `dig` | `dig <domain> <record-type>` | Detailed DNS queries with TTL |
| `arp` | `arp -a` | View ARP cache (IP-to-MAC) |
| `ssh` | `ssh user@host` | Remote secure shell access |
| `wireshark` | GUI | Capture and inspect packets |
